# ACVP Conformance Matrix

This document is based primarily on the official NIST ACVP Protocol Specification and the NIST ACVP ML-DSA JSON Specification. The current repository through Phase 2-9 is a local demo/prototype, not a production ACVP server. Phase 3-1 through Phase 3-5 add documentation, a formal namespace skeleton, local ML-DSA capabilities negotiation, deterministic local vector generation, and an in-memory local state machine only; they do not claim production-ready ACVP server conformance.

Official references:

- ACVP Protocol Specification: https://pages.nist.gov/ACVP/draft-fussell-acvp-spec.html
- ACVP ML-DSA JSON Specification: https://pages.nist.gov/ACVP/draft-celi-acvp-ml-dsa.html
- ACVP documentation landing page: https://pages.nist.gov/ACVP/
- FIPS 204: https://csrc.nist.gov/pubs/fips/204/final

Status values used in this matrix: `SUPPORTED`, `PARTIAL`, `MISSING`, `LOCAL_DEMO_ONLY`, `NOT_IN_SCOPE_YET`, `NEEDS_SPEC_REVIEW`.

The existing `/api/demo/acvp/...` routes are a local demo lifecycle and are not formal ACVP endpoints. The new `/acvp/v1/...` routes are skeleton endpoints under a formal namespace, not a production-ready server. Phase 3-4 vector generation and Phase 3-5 lifecycle state are deterministic/local-fips204-skeleton behavior. A formal production ACVP server still needs DB persistence, authentication/JWT/mTLS, production registration negotiation hardening, durable audit logging, security hardening, and interoperability testing.
